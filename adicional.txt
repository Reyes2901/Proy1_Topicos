# 1️⃣ Concepto de Microservicios

En lugar de tener **un solo proyecto grande** (`main.py` con todos los módulos), cada módulo se convierte en **un microservicio independiente**:

| Módulo         | Microservicio       | Base de datos  |
| -------------- | ------------------- | -------------- |
| Clientes       | `clientes-service`  | `clientes_db`  |
| Productos      | `productos-service` | `productos_db` |
| Ventas         | `ventas-service`    | `ventas_db`    |
| Detalle Ventas | `detalle-service`   | `detalle_db`   |

Cada microservicio:

* Tiene **su propia API**
* Tiene **su propia base de datos** (opcionalmente compartida, pero lo ideal es independiente)
* Puede desplegarse y escalarse **independientemente**
* Se comunica con otros servicios mediante **HTTP / REST o mensajería**

# 3️⃣ Configuración de cada microservicio

## Ejemplo: `clientes-service`

### `database.py`

```python
from sqlalchemy import create_engine
from sqlalchemy.orm import sessionmaker, declarative_base

DATABASE_URL = "postgresql://postgres:password@localhost:5432/clientes_db"

engine = create_engine(DATABASE_URL)
SessionLocal = sessionmaker(autocommit=False, autoflush=False, bind=engine)
Base = declarative_base()
```

### `models.py`

```python
from sqlalchemy import Column, Integer, String
from database import Base

class Cliente(Base):
    __tablename__ = "clientes"
    id = Column(Integer, primary_key=True, index=True)
    nombre = Column(String)
    telefono = Column(String)
    email = Column(String)
```

### `schemas.py`

```python
from pydantic import BaseModel, EmailStr, Field

class ClienteCreate(BaseModel):
    nombre: str = Field(min_length=3, max_length=100)
    telefono: str = Field(min_length=7, max_length=15)
    email: EmailStr
```

### `main.py`

```python
from fastapi import FastAPI, Depends, HTTPException
from sqlalchemy.orm import Session
from database import SessionLocal, engine
import models, schemas

models.Base.metadata.create_all(bind=engine)

app = FastAPI(title="Clientes Service")

def get_db():
    db = SessionLocal()
    try:
        yield db
    finally:
        db.close()

@app.post("/clientes")
def crear_cliente(cliente: schemas.ClienteCreate, db: Session = Depends(get_db)):
    nuevo = models.Cliente(**cliente.dict())
    db.add(nuevo)
    db.commit()
    db.refresh(nuevo)
    return nuevo

@app.get("/clientes")
def listar_clientes(db: Session = Depends(get_db)):
    return db.query(models.Cliente).all()
```

# 5️⃣ Comunicación entre microservicios

Por ejemplo, el **servicio de ventas** necesita verificar un cliente:

```python
import requests

def validar_cliente(cliente_id: int):
    response = requests.get(f"http://localhost:8001/clientes/{cliente_id}")
    if response.status_code != 200:
        raise Exception("Cliente no existe")
    return response.json()
```

> Esto permite que cada microservicio mantenga **su propia lógica y base de datos**, pero se comuniquen mediante **REST**.

* Puedes desplegar cada uno en **un contenedor Docker** en **AWS EC2** o ****
* Se puede añadir **API Gateway** para exponer una URL unificada.
